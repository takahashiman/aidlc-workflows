# Personas — イニシアチブ#2（実開発 dogfooding ＋ ポータル綿密化）

> S2=C により、前サイクル 5 ペルソナ（`personas.md`）を土台に **#2 の2目標へ最適化した専用ペルソナ**を定義。
> 対応関係は各ペルソナ末尾に明記（前サイクルとの連続性を保つ）。

## DEV-A. 既存アプリ改修エンジニア（dogfooding 実行者）★中心
- **アーキタイプ**: 実運用中の製品（BusDelayAlerts / LLocana）を担当し、本システム（FIG-UDS）で**スタイルを整える**エンジニア。
- **目的(Goals)**: 既存機能を壊さず、古い内蔵 DS を FIG-UDS フローへ置換／整理。Core を submodule で pin して再現性を確保。
- **課題(Pain points)**: Tailwind v4 + shadcn と Core 三層の共存、生 HEX 直書き 379 の解消、Vite での配布成立、概念を損なわない範囲の判断。
- **技術文脈**: Vite/React18/Tailwind v4、git submodule、各チーム標準の AI アシスタント、三層ガードレール。
- **主な関与**: Epic D 全般（配布・signature 集約・状態色 semantic 化・@theme ブリッジ・プロファイル）。
- **前サイクル対応**: P2 エンジニア ＋ P3 移行担当 を「既存アプリ改修」に統合・特化。

## UX-D. UX 改修担当（画面遷移・VSCode×Pencil）※S3=C で新設
- **アーキタイプ**: 既存機能を壊さず**画面操作感（UX）**まで改善する担当。画面遷移図を扱う。
- **目的**: 「あわよくば」フロー＝スタイルに加え画面遷移/操作感の改善を、VSCode×Pencil で確立する。
- **課題**: 機能非回帰を保ちつつ UX を変える線引き、デザインツール（Pencil .pen）と実装の往復。
- **技術文脈**: VSCode、**Pencil（.pen 設計ファイル）**、画面遷移図、FIG-UDS スタイル。
- **主な関与**: Epic X の UX 改修フロー確立（US-X2）。
- **前サイクル対応**: 新設（前サイクルに相当なし。画像02「あわよくばポイント」由来）。

## MAINT. Core Maintainer（昇格レビュー・ガバナンス）
- **アーキタイプ**: FIG-UDS Core の番人。昇格判断と taxonomy ガバナンスの最終責任者。
- **目的**: LLocana のドメインパターン（arrival-card/delay-banner 等）の **Core 昇格を実行**（提案→レビュー→マージ→昇格確認）。
- **課題**: 普遍化の手間、他スタイル混入の防止、昇格基準（FIG-UDS 発展途上で基準が育ち中）。
- **技術文脈**: GitHub Issue/PR レビュー、FIG-UDS Live Preview、トークン/CSS 読解。
- **主な関与**: Epic X の Core 昇格実行（US-X1）。
- **前サイクル対応**: P1 Core Maintainer を踏襲。

## VIEW-S. ポータル単独利用の招待エンジニア（セルフ検証）★goal②中心
- **アーキタイプ**: ポータルを**見るだけ**で意図する操作を完遂したい利用者。今回は**ユーザー本人がセルフで代行**。
- **目的**: 役割別入口から、主要4操作（セットアップ/移行/昇格提案/バージョン参照）を口頭補足なしに辿り切る。
- **課題**: 情報過多で目的に辿り着けない、手順の前後関係が不明、GitHub 操作の手順不足。
- **技術文脈**: ブラウザ（公開ポータル / GitHub Pages）、GitHub 基本操作。
- **主な関与**: Epic P 全般（役割別入口・2シナリオ別フロー・オンボーディング・閲覧余白・セルフ検証）。
- **前サイクル対応**: P5 ポータル閲覧者 を「ポータル完結（セルフ検証）」へ特化。

## AI-AG. AI エージェント（各チーム標準: Cursor / Claude 等）
- **アーキタイプ**: ガイド＋開始宣言例に従い作業を代行する非人間アクター。
- **目的**: ポータルの Developer ガイド／テンプレートに従い、ほぼ自動でスタイル修正を進める。三層を遵守。
- **課題**: コンテキスト過多、規約逸脱、ハルシネーション。
- **技術文脈**: スコープ分離 repo、常時コンテキスト規約、開始宣言テンプレート。
- **主な関与**: Epic D/P の自律実行支援、記録（dev-flow-journal/session-log）。
- **前サイクル対応**: P4 AI エージェント を踏襲。
