## 実行シーケンス
```text
        ┌─────────────── dogfooding 系（Critical Path）───────────────┐
U2-1 ──► U2-2 ──► U2-3 ──────────────► U2-6（昇格・蓄積後）
Core      製品     製品                 Core⇄製品
  └────────────────► U2-7（UX改修・U2-2 後に並行可）

        ┌─────────────── ポータル系（並行）───────────────┐
U2-4 ──► U2-5
ポータル  ポータル
```
- **Critical Path**: U2-1 → U2-2 → U2-3 →（蓄積）→ U2-6。
- **並行**: U2-4→U2-5（ポータル）は U2-1〜3 と独立に進行可（U3=A）。U2-7 は U2-2 後に並行可。
- **横断記録**: 全 Unit で dev-flow-journal/session-log を更新（C-Record 内包）。

## repo 跨ぎ調整ポイント
- **U2-1（Core 変更）**: C-Palette を Core へ。Core CI（三層・a11y）が緑であること。昇格 U2-6 と同じ Core repo を触るため、変更を整理（パレット基盤 → 昇格は別 PR）。
- **U2-2（submodule pin）**: U2-1 を含む Core タグに pin（または検証中はブランチ参照→確定後タグ pin）。`CORE-DS-VERSION` 整合。
- **rolling**: ポータル（U2-4/5）は Core content を rolling 取込。U2-1 の Core 変更後に反映され得る。

## リスク・ロールバック
- **U2-1**: a11y AA 不合格時は生成規則を再調整（採用不可ルール）。Core 変更は PR で取消容易。
