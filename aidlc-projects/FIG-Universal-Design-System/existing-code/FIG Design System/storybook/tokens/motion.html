Motion

アプリ全体で「同じ手」が動かしているように感じさせるための motion トークン。cubic-bezier() 直書きは禁止。

Variants — Easing Curves

TokenBezier用途
--easing-standard (0.4, 0.0, 0.2, 1) 既定。等価な「行って戻る」
--easing-decelerate (0.0, 0.0, 0.2, 1) 出現、到着
--easing-accelerate (0.4, 0.0, 1.0, 1) 消滅、退去
--easing-emphasis (0.2, 0.0, 0, 1.2) モーダル登場の overshoot

States

禁止モーション： bounce / spring 系、無限点滅、color hue rotation、durations 直書き (transition: 200ms ease)、independent cubic-bezier()。交通 UX は「信頼性」が第一、誇張は不誠実に感じられる。

Spacing — Duration Ladder

Range用途
40–80ms press / tap 即時反応
120–200ms hover / focus / tab switch
240–320ms enter / exit / modal
400–600ms 大きい layout transition、ページ遷移
800ms+ 原則使わない（待たされる感が強い）

Accessibility

✅
prefers-reduced-motion: reduce 対応： 全 motion token は自動的に near-instant (1ms) に置換される設計。
✅
視差効果は使わない。 Bottom Sheet は単純な translateY のみ、parallax 禁止。
✅
無限ループのアニメは原則禁止（arriving 状態の breathing 等、明確な意味があるもののみ）。
✅
vestibular disorder 配慮： 大きな translate / scale 変化を伴う遷移は prefers-reduced-motion で必ず単純フェードに置換。
