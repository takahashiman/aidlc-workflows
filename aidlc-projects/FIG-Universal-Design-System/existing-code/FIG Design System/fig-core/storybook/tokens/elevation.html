Elevation

「材料としての影」を Primitive 層 (xs〜2xl) で定義し、「役割としての浮き」を Semantic 層 (flat / card / raised / floating / modal / toast / dragged) で参照する 2 層構造。

primitives.css + semantic.css

Spacing

影は周囲に余白を取って初めて視認できる。要素間 spacing と密接に関連する。

Elevation推奨周囲余白理由
flat --space-2 境界線で十分
card --space-3 1–3px の影を切らない
raised --space-4 4–12px の広がりを確保
floating --space-6 30px の広がりを切らない
modal --space-8 視覚的な層分離を強調

Accessibility

✅
影に意味を持たせない： 浮きは 視覚的な強調 のみ。情報の有無や状態は別の手段（色・アイコン・テキスト）で伝える。
✅
ダークモード： ダークテーマでは shadow を弱め、border を強める。Semantic 層で自動切替。
⚠
過剰なレイヤリング： 同一カード内で 2 段以上の elevation ジャンプは避ける（情報階層が崩れて見える）。

Motion

Elevation 変化は必ず motion を伴う。影の hue ジャンプは禁止。

TransitionToken
card → raised (hover) --motion-hover (160ms)
flat → modal (open) --motion-modal-enter (280ms / decelerate)
modal → flat (close) --motion-modal-exit (240ms / accelerate)
card → dragged --motion-press (80ms)
