Route Card

運行情報タブの主役。ユーザー自身が乗車地 → 降車地を編集し、次の便の ETA・運行状況・出発/到着時刻・接近通知をまとめた、本アプリ最大の情報単位。最大 6 枚スタック。

pattern: patterns/route-card (new) impl: <RouteCard> in BusappComponents.jsx arrival-card.md と統合検討

Anatomy

3 段構成。上段「あと N 分」＋ StatusPill ── 状況の一行サマリ。中段 StopSelector 行（O→D ＋ 系統 / のりば） ── 路線設定。下段 時刻表行 ＋ 通知設定 affordance。

Variants — State Mapping

状態は ETA 数字の色 + StatusPill + カード地 の 3 トラックで強化する（色だけに頼らないため pill にもアイコン同梱）。地のティントは弱め（≒ 95% 白）に留め、白地と並んでも違和感のない明度差にする。

Stack — Home List

画像と同様の縦スタック。最後のカードを埋めると新しい空カードが自動で末尾に追加される。最大 6 枚。最終便終了後は ended カードに置換し、フッタの「+ 乗車する停留所を設定する」で追加 row をいつでも開ける。

Accessibility

✅
集約 aria-label：「16系統 渋谷駅前から六本木、あと4分、1番のりば、通常運行、01時04分発」をカード root に。子要素は aria-hidden。
✅
取消線： <s aria-label="変更前 00:28"> + 隣の新時刻に aria-label="変更後 00:43"。色 + 取消線 + ラベルの 3 重で意味を伝える。
✅
状態変化： normal → possible → delayed の遷移は aria-live="polite"。arriving は assertive。
✅
Pencil / 通知 / × のヒット： 個別 button、最小 28×28 でも周囲に 8px 余白を取り 44px の touch target を担保する。
⚠
カードティント： delayed の bg #FEF1F1 上の白カード内文字は AA を維持。red on red になりやすい組合せ（pill 色 / ETA 数字）は十分なコントラストを確認。
