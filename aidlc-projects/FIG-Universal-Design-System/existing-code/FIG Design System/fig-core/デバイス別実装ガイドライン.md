エンジニアにとって最も「フレンドリー」な管理方法は、「何を使うか（Semantic Token）」を変えずに、「その値が何であるか（Value）」だけをデバイス文脈（Profile）に応じて切り替える設計にすることです。具体的に、ご提示いただいた3つのパターンを効率的に管理するための「デバイス・プロファイル」戦略を提案します。

1. デバイス・プロファイルによる「値の流し込み」エンジニアがコードを書く際、font-size: 16px と直接書くのではなく、var(--fig-size-body) という「意味（トークン）」を使います。このトークンの実数値を、プロジェクトのプロファイルに応じて自動で切り替えます。3つの表示パターンの定義パターンプロファイル名特徴最適化の方向性① 特定モバイルMobile-Terminal特定の端末（無線機など）に固定固定視認性: 端末の画面比率に最適化した固定値。② PCメインWeb-Admin管理者画面 (BtoB)情報密度: 一画面に多くの情報を載せるため、小さめの文字と緻密な余白。③ スマホメインMobile-Consumerユーザー向けアプリ (CtoC)操作性（慈愛）: 親指で押しやすい大きなボタン（44px以上）と高い可読性。2. エンジニアフレンドリーな構成案CSSの構成を以下のように分離することで、エンジニアは「自分のプロジェクトに該当するプロファイル」を読み込むだけで済みます。ディレクトリ構造例

このように、class を一つ変えるだけで FIG の理念（スピード・慈愛）に基づいた最適解が適用される状態が、最もエンジニアフレンドリーです。次のステップ：デバイス別スタイルの具体値定義この構成で進めるにあたり、まずは 「標準的なフォントサイズ（Body）」 や 「ボタンの高さ」 を、各プロファイルで具体的にどう変えるかの「数値定義表」をポータルサイトに掲載するためのドラフトを作成しましょうか？

\# FIG Universal Design System - デバイス別実装ガイドライン



\## 1. 3つの表示パターンの定義



エンジニアがプロジェクトの性質に応じて選択すべき「デバイス・プロファイル」を以下の3つに定義します。



| パターン | プロファイル名 | 最適化の方向性 | 経営理念との連動 |

| --- | --- | --- | --- |

| \*\*① 特定モバイル\*\* | `Mobile-Terminal` | 無線機端末等の固定画面サイズに最適化した視認性。 | \*\*俯瞰\*\*: 端末の特性を考慮した専用設計。 |

| \*\*② PCメイン\*\* | `Web-Admin` | 管理画面等の情報密度を優先したBtoB向け設計。 | \*\*スピード\*\*: 情報の一覧性を高め、業務効率を最大化。 |

| \*\*③ スマホメイン\*\* | `Mobile-Consumer` | 一般ユーザー向けの操作性とアクセシビリティを重視。 | \*\*慈愛\*\*: 誰一人取り残さない直感的なUI。 |



\## 2. エンジニアフレンドリーなディレクトリ構成



「不変の基盤（Core）」と「可変の数値（Profile）」を分離し、エンジニアが最小限の読み込みで実装を完了できる構造にします。



/design-system
  ├── tokens/
  │    ├── base.css           /* 全社共通（FIGカラー、不変のブランド定義） */
  │    ├── profile-terminal.css  /* パターン①：特定モバイル用の上書き */
  │    ├── profile-admin.css     /* パターン②：管理者PC用の上書き */
  │    └── profile-consumer.css  /* パターン③：一般ユーザーアプリ用の上書き */
  └── components/
       └── buttons.css        /* セマンティックトークン（変数）を用いた部品定義 */


\### 実装の仕組み



エンジニアは `index.html` の `<body>` タグ等にクラスを付与するだけで、UIの最適化を完了させます。



<body class="fig-profile-consumer">
  <button class="fig-button">送信</button>
</body>



\## 3. ポータルサイト（index.html）での管理方法



ポータルサイト自体を「開発者が迷わないための俯瞰的なツール」として機能させます。



\* \*\*ライブプレビュー・スイッチ\*\* 🕹️

\* コンポーネントの表示エリアに「Admin」「Consumer」「Terminal」の切り替えボタンを配置します。

\* スイッチを切り替えると、iframe内の表示クラスが動的に変更され、デバイスごとの挙動差（文字サイズやボタンの高さ）を即座に確認できます。





\* \*\*コピペ可能なスニペット\*\* 📋

\* 選択したプロファイルに応じた「HTML/CSSの実装コード」を表示します。

\* 「どの変数を、どのプロファイルで使うべきか」を明示し、開発のスピードを加速させます。





\* \*\*アクセシビリティ・チェックリスト\*\* ✅

\* DADS（デジタル庁）の基準をベースに、プロファイルごとに遵守すべきアクセシビリティ項目（タップターゲットの大きさ等）を表示し、慈愛の精神をコードに落とし込みます。
