# バスアプリ デザインシステム

**バス遅延情報サイト** から抽出したデザインシステムです。京都府・兵庫県・大分県・沖縄県の通勤者向けに、バスの到着時刻・遅延状況の確認、および（京都府のみ）デジタル定期券による NFC タッチ決済を提供する、モバイルファーストの日本語 Web アプリです。

## 出典

- **GitHub リポジトリ:** `takahashiman/BusDelayAlerts`（プライベート）。参照したコミット: `0c38ec98`。
- **Figma 原本:** https://www.figma.com/design/kZEOQ1T4KKnbSRgmtNmVIn/%E3%83%90%E3%82%B9%E9%81%85%E5%BB%B6%E6%83%85%E5%A0%B1%E3%82%B5%E3%82%A4%E3%83%88
- 技術スタック: Vite + React 18 + Tailwind v4 + shadcn/ui + Radix プリミティブ + lucide-react アイコン + `motion`（Framer Motion）。

このドキュメントを読む方が上記リソースにアクセスできるとは限りません。**この README を単一の正となる情報源として扱ってください。** 必要な情報はすべてこのデザインシステム内にインライン化しています。

## プロダクト概要

本アプリは単一プロダクトのモバイル Web 体験で、主要な画面が 2 つあります。

1. **運行情報** — デフォルトのタブ。「ルートカード」を縦に積み重ねたリストで、各カードでユーザーが乗車地と降車地を選択すると、次のバスの到着時刻、運行状況（通常運行 / 遅延の可能性 / 遅延確定 / 通過済）、およびカードごとに設定可能な接近通知ルールが表示されます。カードは出発時刻順に自動ソートされ、最後のカードを埋めると新しい空のカードが追加されます（最大 6 枚）。
2. **乗車・決済** — 京都府のみ。デジタル定期券カードに 3 つの状態（有効 / 乗車中 / 期限切れ）があり、NFC タップのシミュレーション（iPhone / Android 切替）、乗車 → 降車のフロー、運賃サマリーモーダルを備えます。

その他の画面: オンボーディング（地域選択）、サイドバードロワー（アカウント / 地域 / 設定 / ログアウト）、地域設定、プロフィール、利用規約、エラー、通知設定、停留所のマップ検索。地域選択は実在する 4 県（京都府・兵庫県・大分県・沖縄県）＋ スキップ状態に対応しています。地域は `localStorage` に永続化されます。

想定ユーザーは日本の通勤者で、**UI コピーはすべて日本語** です。トーンはですます調（謙譲表現「申し訳ございません」を含む）。状態表示やエラーメッセージは謝罪と安心感を重視しています。時刻は JST、形式は `YYYY/MM/DD HH:MM`。通貨は `¥`。

## コンテンツの基本（CONTENT FUNDAMENTALS）

**言語:** 日本語のみ。ラベル、コピー、エラー、マイクロコピーすべて。

**トーン:** ですます調・やや謝罪寄り・安心感を与える文体。公共交通の「お知らせ」に近く、おしゃべりな消費者向けアプリではありません。**絵文字は使用しません。** 「私たち vs あなた」のフレーミングも避け、アプリ自身ではなくバスについて第三者的に語ります。

**表記:** 日本語にケースの概念はありませんが、ラテン文字を使う固有名詞・略語は **ID / NFC / iPhone / Android** など正式表記を守ります。数字は **半角**（210、`２１０` ではなく）。

**実コピー例（ソースコードより抽出）:**

| 場面 | コピー | メモ |
|---|---|---|
| ステータス（通常） | `通常運行` | 4 文字、句読点なし |
| ステータス（遅延） | `約15分遅延` | 「約」で精度を和らげる |
| ステータス（運休） | `運休` | 簡潔・事実ベース |
| 遅延理由 | `事故の影響により、遅延が発生しております。ご迷惑をおかけして申し訳ございません。` | 完全な謝罪文・謙譲形 |
| 雪による遅延 | `降雪の影響により、大幅な遅延が発生しております。` | 理由 → 影響 → 規模 |
| 運休理由 | `車両故障のため、運休しております。復旧まで今しばらくお待ちください。` | 必ず次の見通しを提示 |
| 定期券ヘッダ（待機） | `定期券 有効（乗車前）` | 括弧で副状態を補足 |
| 定期券ヘッダ（乗車中） | `乗車中` | 簡潔・点滅ドットと併用 |
| タップ案内（iPhone） | `端末の先端を近づけてください` | デバイス別文言 |
| タップ案内（Android） | `端末の背面をかざしてください` | 「先端」→「背面」に差し替え |
| 注意書き | `※ スマホケースに厚みがあると反応しない場合があります` | 注意書きは先頭に `※` |
| デモ用注釈 | `※ タップで乗降シミュレート` | デモ・開発用注釈も `※` |
| 完了モーダル | `ありがとうございました` | 「成功」ではなく感謝の言葉 |
| 失敗モーダル | `読み取りに失敗しました` / `もう一度お試しください` | 事実 + 再試行の案内 |
| サイドバー（ゲスト） | `ゲストユーザー` / `アカウント未登録` | 控えめな未登録表現 |

**ライティングのパターン:**
- 遅延メッセージは「理由 → 影響」の順: `〈事象〉の影響により、遅延が発生しております。`
- 大きな遅延・運休は謝罪のペアで締める: `ご迷惑をおかけして申し訳ございません。`
- `※` は注意書き・デモ注釈に使用。ユーザー必須事項には使わない。
- サイドバーのセクション見出しはラテン大文字で `tracking-wider`（`ACCOUNT`, `SETTINGS`）。直下の項目ラベルは日本語。
- 数字は常に半角。円記号は `¥`。
- 時刻表記: ログ等は `2026/03/11 14:23`、フロー内は `14:23`。

## ビジュアルの基本（VISUAL FOUNDATIONS）

### 色のムード

抑えた **FIG ターコイズ** のプライマリカラー（`#26B7BC`）に、**やわらかなFIG ターコイズ・ブルーのグラデーション** 背景を組み合わせます。背景は `linear-gradient(135deg, #EFF9FA 0%, #F4F8FB 50%, #EBF4FB 100%)` ── ほとんど目立たない、グラデーションというよりは色のウォッシュです。カードは純白で背景に浮かびます。ステータス色は Tailwind デフォルト（green-50/700, amber-50/700, red-50/700, orange-500）を流用。定期券カードのみ濃いティール → エメラルドのグラデーション（`#26B7BC → #5CCED2 → #1A8589`）── アプリ内で唯一の彩度の高い面です。コードベース内の画像はスレート系のラインイラスト・プレースホルダーのみで、写真は同梱されません。

### タイポグラフィ

**フォントスタック:** Tailwind デフォルトの `font-sans`。コードベースには **カスタム Web フォントは含まれず**、OS の日本語システムスタック（iOS / macOS の Hiragino Sans、Windows の Yu Gothic / Meiryo、Android の Noto Sans JP）に依存します。デザインシステムのプレビューでは表示の一貫性のために Google Fonts の **Noto Sans JP** で代替しています。⚠️ **要確認:** リポジトリにフォントファイルがないため、iOS Safari と完全に同じ表示が必要な場合は Hiragino Sans のローカル設置が必要です（無料の完全互換フォントはありません）。

**サイズスケール（ソースコードより抽出）:**
- 見出し / ヒーロー: `28px`–`32px` extrabold（定期券カードの「区間」）。
- セクションタイトル: `text-lg` bold（「バスアプリ」ヘッダー）。
- 本文: `text-sm` / `14px` regular〜medium。
- マイクロコピー: `text-xs` / `11px`。
- ステータスバッジ: `text-sm font-medium`。
- セクションラベル: `text-xs font-bold uppercase tracking-wider`（サイドバーのみ）。

ウェイト: 400 / 500 / 600 / 700 / 800（800 は定期券カードのヒーローテキスト専用）。

### スペーシング

- ページ余白: `px-4 py-5`（モバイル基準）。
- カード内側: `p-4`（ルートカード）/ `p-5`–`p-6`（定期券カード）。
- カード間の縦間隔: `space-y-4`。
- アイコンとラベルの密な間隔: `gap-1.5` 〜 `gap-3`。
- ヘッダー高さ: `py-3`（およそ 52px）。

### 背景

- アプリ背景: 3 ストップのパステルミントグラデーション（`#EFF9FA → #F4F8FB → #EBF4FB`）、`bg-gradient-to-br`。ほぼ知覚できないレベルで、わずかにクール寄りのオフホワイトのよう。
- ホーム画面の装飾ブラー: `w-64 h-64 bg-white/40 rounded-full blur-3xl` と `bg-[#26B7BC]/5`。`-z-10` でコンテンツの背後に置き、深みを出すが主役にはしない。
- ヘッダー: `bg-white/70 backdrop-blur-lg border-b border-white/60` ── スティッキーなフロステッドガラス。
- モーダル: `bg-black/25` 〜 `bg-black/40` のオーバーレイ + `backdrop-blur-sm`。

**全画面写真・手描きイラスト・繰り返しパターン・テクスチャは使用しません。** 背景は色のウォッシュ + ぼかしブロブのみです。

### アニメーション

ライブラリ: `motion/react`（Framer Motion）。
- **イージング:** `[0.22, 1, 0.36, 1]`（カスタム expo-out）── カード・モーダル登場時。`easeInOut` ── 呼吸・シマー。スプリング `{stiffness: 200, damping: 25}` ── サイドバーのスライド。
- **継続時間:** UI トランジション 0.25–0.5 秒。呼吸パルス 1.2 秒。NFC リング拡張 2.0–2.4 秒。定期券シマー 3.0 秒。
- **パターン:** 登場時の fade + y（`opacity 0→1`、`y ±8〜±30`）、モーダルは `scale 0.85→1`、NFC タップエリアでは 1.2 秒ごとにパルスリングを生成、`乗車中` バッジには呼吸ドット。
- **ホバー:** `hover:bg-slate-50`（明るい塗り）、`hover:bg-[#26B7BC]/10`（プライマリ色のティント）、`hover:bg-amber-50`（警告寄りの暖色）。ボーダーはホバーでほぼ変えない。
- **プレス:** カード・ボタンに `active:scale-[0.98]`、NFC タップターゲットには `whileTap={{ scale: 0.92 }}`。
- バウンス・オーバーシュート・派手な動きはなし。すべて素早く落ち着く。

### ボーダー / 角丸 / シャドウ

- **角丸:**
  - ボタン・入力: `rounded-md`（6px、shadcn デフォルト）/ `rounded-xl`（12px、アプリ内ボタン）。
  - カード: 標準 `rounded-xl`（12px）、タブバー `rounded-2xl`（16px）、ヒーロー / 決済面はカスタム `rounded-[20px]`・`rounded-[28px]`。
  - ピル / ステータスバッジ: `rounded-full`。
- **ボーダー:** ほとんど `border-slate-100` / `border-slate-200`、1px。定期券カードは `border-white/15`。入力は `border-2 border-gray-300`。
- **シャドウ:**
  - カードホバー: `hover:shadow-lg`。
  - ヘッダー: `shadow-[0_4px_30px_rgba(0,0,0,0.03)]` ── ごく薄い。
  - 定期券カード: `shadow-[0_20px_50px_rgba(38, 183, 188,0.25)]` ── 色付き・深い。
  - アクティブタブ: `shadow-[0_2px_16px_rgba(0,0,0,0.06)]`。
  - FAB: `shadow-[0_8px_30px_rgba(0,0,0,0.12)]`。
- インナーシャドウ・`box-shadow: inset` は使用しません。

### 透過とブラー

意図的に多用します。
- ヘッダー / タブバー / モーダルオーバーレイ: `backdrop-blur-sm` 〜 `backdrop-blur-lg`。
- 装飾ブロブ: `blur-3xl` で大きなソフトグロー。
- 半透明の塗り: `bg-white/40`（装飾）、`bg-white/70`（ヘッダー）、`bg-slate-200/50`（タブバー背景）、`bg-[#26B7BC]/10`（プライマリティント）。
- モーダルのスクリム: `bg-black/25–40 backdrop-blur-sm`。

**フロステッドガラス + 半透明ティント** がこのブランドで最も識別しやすいビジュアル要素です。

### レイアウトのルール

- モバイルファースト。最大コンテンツ幅 `max-w-3xl`（およそ 768px）、`mx-auto` で中央寄せ。
- ヘッダーは上部固定、コンテンツがその下をスクロール。
- 状態プレビュー切替の FAB は `bottom-6 right-6` に固定。
- 下部のセーフエリア: ルートに `pb-safe`。
- サイドバー: `w-72` の左ドロワー、`bg-slate-900/40 backdrop-blur-sm` のスクリム上にスライドイン。
- モーダル: 中央配置、`max-w-sm`、ビューポートから `px-6` のマージン。

### カード

- 標準カード: `bg-white rounded-xl border border-gray-200 p-4 hover:shadow-lg active:scale-[0.98] transition-all`。
- 定期券カード: 濃いティールグラデーション、`rounded-[28px]`、ボーダーは `border-white/15` のみ、色付きの厚いシャドウ。
- タブバー: `bg-slate-200/50 p-1.5 rounded-2xl backdrop-blur-sm border border-slate-200/50` ── ピル型のセグメンテッドコントロール。アクティブセグメントは白で塗り、非アクティブは `text-slate-500` にフェード。

### このデザインシステムが避けるもの

- 青紫系のグラデーション（ティール × エメラルドのみ使用）。
- 絵文字。
- 手描き風 SVG イラスト・生成画像。
- 角丸 + 左ボーダーだけ色付きのカード（生成 AI デザインによくある癖）。
- 重く大袈裟なシャドウ ── 重要性を示す定期券カードを除く。

## アイコノグラフィ

コードベースは **lucide-react** のみを使用します。カスタム SVG イラスト・アイコンフォント・PNG アイコンはありません。ソースから抽出したアイコン: `User`, `Settings`, `Bell`, `X`, `LogOut`, `ChevronRight`, `ChevronDown`, `Ticket`, `MapPin`, `Bus`, `Clock`, `CheckCircle`, `XCircle`, `Info`, `Smartphone`, `Nfc`, `ShieldCheck`, `ShieldOff`, `AlertTriangle`, `HelpCircle`, `ArrowLeftRight`, `CalendarDays`, `Hash`, `Apple`, `MonitorSmartphone`, `Check`, `RotateCcw`, `CircleDollarSign`, `RefreshCw`, `Route`, `CreditCard`, `LayoutTemplate`, `Map`, `Menu`。

**サイズ:** インラインアイコン `size-3.5`（14px）、ステータスバッジ・リスト行 `size-4`（16px）、セクション見出し `size-5`（20px）、タブ・メニュー主要 `size-6`（24px）、NFC ヒーローアイコン `size-12`（48px）。

**ストローク:** lucide デフォルト（2）。ヘッダーのメニューアイコンは明示的に `strokeWidth={2}` を指定。

**カラー:** アイコンは `currentColor` を継承。多くの場合、同系色の背景バブル（`bg-blue-100 text-blue-700 rounded-xl p-2.5` など）と組で使用。

**絵文字:** 一切使いません。

**Unicode 文字をアイコンとして使う:** 定期券カードの区間表示で `↔` のみ（`渋谷駅 ↔ 六本木駅`）。それ以外では使いません。

このデザインシステムでは lucide を CDN（UMD グローバル `lucide`）からインライン SVG として読み込みます。アセットを別途コピーする必要はありません ── 別アイコンセットへの差し替えはむしろ後退になります。

## 既知の制約・代替対応

- **ソースにフォントファイルがありません。** アプリは OS の日本語システムスタックに依存しています。プレビューの一貫性のため Google Fonts の Noto Sans JP で代替しています。⚠️ iOS Safari との完全一致が必要な場合は Hiragino Sans を別途ご用意ください。
- **ソースにロゴアセットがありません。** ヘッダーは `バスアプリ` のテキストワードマークのみ。`assets/logo.svg` として再現しています。
- **オンボーディング画面は深掘りしていません。** ソースには `Onboarding.tsx` が存在しますが、UI キット内では可視状態の忠実な近似のみ ── マルチステップフローの完全再現ではありません。
- **マップ検索 / 路線詳細 / 設定 / 通知設定** ── ソースでは独立した完全なページとして存在しますが、UI キットでは要約的な単一画面で再現しています（オリジナルはそれぞれ 15k〜38k バイト）。
