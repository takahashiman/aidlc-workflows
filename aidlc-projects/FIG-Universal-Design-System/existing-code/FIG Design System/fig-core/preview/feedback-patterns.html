Feedback Patterns

視覚 (motion) + 触覚 (haptic) + 聴覚 (audio) の三位一体規約を semantic.css §20 から呼び出して実演する。振動非対応・音声 OFF・prefers-reduced-motion いずれも各チャネルが個別に no-op するフォールバックを確認できる。

1. 現在の環境

ブラウザ / OS の機能検出と CSS 変数の現在値を表示。緑＝有効、灰＝無効 / 未対応。

確認方法：モバイル端末で開くと navigator.vibrate が「対応」になり、各 Feedback ボタンを押すと振動する。OS で「動きを減らす」を ON にすると --haptic-enabled: 0 が立ち、視覚も振動も即時停止に切替わる。

3. ライブ実演

ボタンを押すと、対応する Feedback ID を fireFeedback() で発火。視覚は keyframes、触覚は navigator.vibrate()、音声は --audio-cue-enabled: 1 のときだけ鳴る。

cta-firemedium haptic + ui-tap
—
cta-successsuccess haptic + ui-success
—
cta-errorerror haptic + ui-error
—
togglelight haptic / 音なし
—
selectlight haptic / 音なし
—
nfc-touchheavy haptic + nfc-touch
—
視覚チャネルはこのデモでは tick / bounce / shake の 3 種の keyframes を割り当てている（実装では各コンポーネントの transition に直接 motion トークンを流す）。
触覚チャネルはモバイルブラウザ（Chrome on Android 等）でのみ発火。デスクトップでは log に「振動なし」と表示。
音声チャネルは既定 OFF。上の toggle で ON にすると音源を読みに行くが、本プロジェクトに音源は同梱されていないため 404 で no-op する（contract のみ検証）。
