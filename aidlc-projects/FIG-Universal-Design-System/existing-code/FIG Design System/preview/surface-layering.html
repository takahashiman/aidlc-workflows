SURFACE LAYERING

背景 ＞ カード ＞ 状態カード ＞ 浮きカード

Surface 中心 UI の階層を可視化。プレーンな白カードの上にネストする内部ブロックや、ステータス・浮き要素の使い分けを示します。

▽ color.surface.canvas — ページ最下層（このフレームの背景）
surface.default
標準カード（白）。ホームのバスカード、サイドバー、モーダル。
surface.container-low #F8FAFC
カード内のサブ領域。リスト行ホバー塗り、入力グループ背景。
surface.container #F1F5F9
状態カード。マッチ済み便のサマリー枠、「通過済」バッジ背景。
surface.container-high #E2E8F0
タブバー非アクティブ背景。さらに一段奥の枠。
+ shadow → 浮きカード
surface.default に shadow-card-hover を当てた状態。FAB、トースト、ドラッグ中。
surface.brand · 決済タブアクティブ / PrimaryButton
surface.success · 通常運行ピル
surface.warning · 遅延の可能性 / 注意バナー
surface.error · 遅延確定 / 運休 / NFC 失敗
使い分けのルール。 白カード（surface.default）にネストする内部ブロックは container-low から始め、強調が必要なら container、さらに奥なら container-high と段階的に濃くします。同じカード内で 3 段以上ネストしないこと。浮きカードは常に surface.default + shadow の組み合わせで、ティント済みサーフェス（brand / warning / error）上には浮かせません。
