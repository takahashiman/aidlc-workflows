複製方法

新規プロジェクト作成手順

  1. extensions/busapp を extensions/[ProjectName] にコピー
  2. project-settings.json を編集：
    • projectName を新規プロジェクト名に変更
    • signatureColor.hex を新規ブランド色に変更
    • harmonization を新規色の説明に変更
  3. ポータルをリロード → 新規プロジェクトが自動登録される
  4. Live Preview で新規色が自動反映されていることを確認

新規プロジェクト追加方法

このポータルに新規プロジェクトを追加するには：

  1. extensions/busapp を複製
  2. project-settings.json を編集
  3. このポータルをリロード

詳細は DEPLOYMENT-GUIDE.md を参照してください。

設計ガイドライン

三層アーキテクチャ

Layer 1: Primitive

生値（HEX色、px、数値）。エンジニア / デザイナー以外は直接参照しない。

例: --color-brand-primary: #26B7BC

Layer 2: Semantic

役割トークン。Primitive を参照し、Component に提供する。

例: --color-surface-brand: var(--color-brand-primary)

Layer 3: Component

React JSX。Semantic のみを参照。直書き禁止。

例: backgroundColor: var(--color-surface-brand)

禁止事項（MUST NOT）

絶対に書かないこと

  • color: #26B7BC → var(--color-text-brand) を使う
  • font-size: 22px → var(--typography-headline) を使う
  • box-shadow: 0 4px 30px → var(--surface-elevation-card) を使う
  • z-index: 50 → var(--z-modal) を使う
  • Tailwind 任意値（text-[22px], bg-[#26B7BC]）

複製テンプレート化の実現

新規プロジェクトで自動色反映を実現するには

  • React コンポーネント JSX が生値を持たない
  • CSS は 100% Semantic トークン参照
  • project-settings.json の色を変えるだけで、すべてのコンポーネントが新色で再レンダリング
  • エンジニアが 1 行もコードを修正しない
